# Smart-Energy-Insight

ขั้นตอนที่ 1: การจำลองและรวบรวมข้อมูล (Data Acquisition & Simulation)
เนื่องจากเราไม่ได้ไปติด Sensor จริง เราจะสร้าง "Synthetic Data" (ข้อมูลจำลอง) ที่สมจริงขึ้นมาเพื่อใช้ในการ Clean และ Train ครับ

สร้างชุดข้อมูล: ใช้ Python (Pandas + Numpy) สร้างตารางข้อมูลรายชั่วโมง (Hourly Data) ย้อนหลัง 1-2 ปี

ตัวแปรที่ต้องมี (Features):

Timestamp: วันและเวลา

Outdoor_Temp: อุณหภูมิภายนอก (ตัวแปรสำคัญที่ทำให้แอร์กินไฟ)

Aircon_Usage: จำนวนชั่วโมงที่เปิดแอร์ หรือระดับอุณหภูมิที่ตั้งไว้

Occupants: จำนวนคนในบ้าน ณ เวลานั้น

Appliance_Load: การกินไฟของเครื่องใช้ไฟฟ้าอื่นๆ (ตู้เย็น, ไมโครเวฟ)

Total_kWh (Target): ผลรวมการกินไฟทั้งหมดที่เราจะพยากรณ์

ขั้นตอนที่ 2: การทำความสะอาดข้อมูล (Data Cleaning - 2 คะแนน)
ขั้นตอนนี้สำคัญมากสำหรับคะแนนส่วนนี้ คุณต้องเขียน Code เพื่อจัดการสิ่งต่อไปนี้:

จัดการค่าที่รวน (Outliers): จำลองว่า Sensor ส่งค่าติดลบหรือค่าที่สูงผิดปกติ (เช่น 10,000 หน่วยใน 1 ชม.) แล้วใช้ Z-score หรือ Clip เพื่อตัดค่าพวกนี้ทิ้ง

จัดการค่าว่าง (Missing Values): สมมติว่า Wi-Fi หลุดทำให้ข้อมูลหายไปบางช่วง ให้ใช้ Interpolation หรือ Forward Fill เพื่อเติมข้อมูลให้ต่อเนื่อง

Feature Engineering: สร้างคอลัมน์ใหม่ เช่น Is_Summer, Is_Weekend, Peak_Hour (18:00-22:00) เพื่อให้ Model เข้าใจว่าช่วงไหนไฟแพงหรือใช้เยอะ

ขั้นตอนที่ 3: การใช้ PyCaret พยากรณ์ (ML Implementation - 3 + 5 คะแนน)
เมื่อได้ข้อมูลที่สะอาดแล้ว ให้ส่งต่อให้คนทำ ML:

Setup: ใช้ pycaret.regression เพื่อเตรียมข้อมูล

Compare Models: รัน compare_models() เพื่อหาว่า Algorithm ไหน (เช่น CatBoost, XGBoost, หรือ LightGBM) พยากรณ์ค่าไฟได้แม่นที่สุด

Finalize: บันทึก Model ที่ดีที่สุดเป็นไฟล์ .pkl เพื่อนำไปใช้ใน Dash

ขั้นตอนที่ 4: สร้าง Dashboard ด้วย Dash (Frontend - 5 + 1 คะแนน)
ส่วนหน้ากากที่ผู้ใช้จะมองเห็น:

Layout: ใช้ Dash Bootstrap Components (DBC) แบ่งหน้าจอเป็น 2 ฝั่ง:

ฝั่งซ้าย (Input): มี Slider ปรับอุณหภูมิแอร์, Dropdown เลือกเครื่องใช้ไฟฟ้า, Input ใส่ค่าไฟต่อหน่วย

ฝั่งขวา (Visualization): แสดงกราฟราคาไฟรายเดือน (Actual vs Predict) และ Gauge Chart แสดงระดับความประหยัด

CSS: ตกแต่งโทนสี "Dark Mode" หรือ "Eco Green" เพื่อให้ดูทันสมัย

ขั้นตอนที่ 5: โมดูลเพิ่มเติม: "Monthly Bill Anomaly Detection" (5 คะแนน)
นี่คือส่วนที่คุณต้องการ คือการดูเฉพาะเดือนที่มีปัญหา:

ฟังก์ชัน: ระบบจะเปรียบเทียบการใช้ไฟของเดือนปัจจุบันกับค่าเฉลี่ยปีที่แล้ว

การแจ้งเตือน: ถ้าเดือนไหนค่าไฟกระโดดผิดปกติ (เช่น แอร์เริ่มเก่าทำให้กินไฟมากขึ้น) ระบบจะแสดงข้อความเตือนว่า "พบความผิดปกติในเดือนนี้: แนะนำให้ล้างแอร์หรือเช็คขอบยางตู้เย็น"

ขั้นตอนที่ 6: การจัดการ Git และ รายงาน (2 + 2 คะแนน)
Git: แบ่ง Folder เป็น data/, notebooks/ (สำหรับ PyCaret), และ app/ (สำหรับ Dash) ให้เพื่อนทุกคน Commit งานอย่างสม่ำเสมอ

รายงาน: สรุปว่า Model ตัวไหนแม่นที่สุด และ Dashboard นี้จะช่วยคนประหยัดไฟได้อย่างไรในระยะยาว
